Escrow Constraints
Escrow Constraints
Description
FinishAfter (Required)
The earliest time (in Ripple Epoch seconds) that the escrow can be finished. Must be in the future.
CancelAfter (Optional but common)
The time after which the escrow can be **canceled** if not yet finished. Must be **after** `FinishAfter`.
Condition (Required for conditional)
A cryptographic condition (usually a hash of a preimage). The escrow can only be finished by submitting the matching `Fulfillment`.
Fulfillment (Required to finish)
The preimage that satisfies the `Condition`. Submitted with the `EscrowFinish` transaction.
EscrowFinish rules
Must include both `Condition` and `Fulfillment` to succeed. Cannot be submitted before `FinishAfter`.
**Only recipient can finish**
The **Destination** of the escrow (recipient) must finish the escrow with the correct `Fulfillment`, and only after `FinishAfter`.
**Only sender can cancel**
The original escrow sender can cancel the escrow, but only **after `CancelAfter`**, and only if it hasn’t already been finished.
Ledger time vs system time
Times are in **Ripple Epoch**, not UNIX. Use XRPL server time for accurate conversions.
Immutable once created
You cannot modify `FinishAfter`, `CancelAfter`, `Condition`, or `Amount` after creation.
XRP-only
Escrow transactions only support **XRP**, not IOUs or issued currencies.
