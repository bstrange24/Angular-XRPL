Description
Limit must be > 0 to establish line
A trust line is only created if the `LimitAmount.value` is greater than zero.
Trust line is removed with value = 0
Setting `LimitAmount.value` to `"0"` and submitting removes the trust line **if balance is also 0**.
Cannot remove with non-zero balance
You cannot remove a trust line if the balance is not 0.
Currency must be 3-letter or hex
Currency codes must be standard 3-letter ISO or 160-bit hex-encoded codes.
No create-credentialss needed for XRP
XRP does **not** require or support trust lines.

create-credentials Constraints
Description
TransactionType
Must be "TrustSet".
Account
The address of the account setting the trust line.
LimitAmount
Defines the trust line: includes currency, issuer, and value (max IOU the account is willing to hold from issuer).
Send Max
**For IOU payments**, limits the amount to spend. *Not needed when sending XRP.*
Deliver Min
Minimum amount that must be delivered. Useful in **partial payments**. *Ignored for XRP payments unless `tfPartialPayment` is set.*
Paths
Array of paths for cross-currency payments. *Not applicable for XRP-to-XRP.*
Flags
Used to enable/disable certain trust line behaviors (see next table).
QualityIn
Optional inbound transfer rate (e.g. for IOU exchange quality).
QualityOut
Optional outbound transfer rate.
